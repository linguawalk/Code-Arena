## 1단계: 컴퓨터 사고 기초 (Computer Thinking)

### 학습 목표
- 문제를 단계별로 분해(decomposition)하는 능력을 기른다
- 알고리즘이 "정확한 순서의 명령 목록"임을 이해한다
- 변수, 조건, 반복이라는 세 가지 핵심 개념을 언어 없이 체득한다

### 핵심 개념
1. **알고리즘** — 문제 해결 절차의 정의
2. **순서도 (Flowchart)** — 조건 분기와 흐름 시각화
3. **변수** — 이름이 붙은 기억 공간
4. **조건문** — "만약 ~라면" 분기
5. **반복문** — 패턴의 반복과 종료 조건
6. **함수 개념** — 재사용 가능한 명령 묶음
7. **입력 / 출력** — 컴퓨터와의 대화 구조
8. **디버깅 사고** — 오류를 찾아내는 논리

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- 순서 배열 (드래그) ← 이 단계의 핵심
- 순서도 완성 (빈칸 채우기)

### 예시 문제

#### [1-1] 순서 배열 — 라면 끓이기 알고리즘
**문제:** 다음 단계를 올바른 순서로 배열하세요.

> A. 물이 끓으면 면을 넣는다  
> B. 냄비에 물을 붓는다  
> C. 면이 익으면 스프를 넣는다  
> D. 불을 켠다  
> E. 그릇에 담는다

**정답:** B → D → A → C → E  
**해설:** 컴퓨터 프로그램도 이처럼 순서가 틀리면 원하는 결과가 나오지 않습니다.

---

#### [1-2] 개념 퀴즈 — 변수의 역할
**문제:** 다음 중 '변수'를 가장 잘 설명한 것은?

> ① 컴퓨터가 실행하는 명령어  
> ② 이름이 붙은 기억 공간으로, 값을 저장하고 바꿀 수 있다  
> ③ 프로그램의 시작점  
> ④ 오류가 발생하는 코드 줄

**정답:** ②  
**해설:** 변수는 우편함과 같습니다. 함함(이름)은 그대로지만 안의 내용(값)은 바꿀 수 있습니다.

## 2단계: C언어 (C Language)

### 학습 목표
- 메모리와 CPU가 실제로 어떻게 동작하는지 코드로 이해한다
- 포인터를 통해 "주소로 생각하기"를 체득한다
- 수동 메모리 관리를 경험하며 컴퓨터의 언어에 가까워진다

### 핵심 개념
1. **컴파일 과정** — 소스코드 → 기계어 변환
2. **자료형** — `int`, `char`, `float`, `double`과 메모리 크기
3. **연산자** — 산술, 비교, 논리, 비트 연산
4. **조건문** — `if / else if / else`, `switch`
5. **반복문** — `for`, `while`, `do-while`
6. **배열** — 연속된 메모리 블록
7. **함수** — 선언, 정의, 호출, 반환값
8. **포인터** — 주소와 역참조 (`*`, `&`)
9. **문자열** — `char` 배열과 null terminator
10. **구조체 (`struct`)** — 사용자 정의 자료형
11. **동적 메모리** — `malloc`, `free`
12. **파일 I/O** — `fopen`, `fprintf`, `fclose`

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- 코드 완성 (빈칸 채우기)
- 출력 예측 (단답형)
- 순서 배열 (함수 실행 흐름)

### 예시 문제

#### [2-1] 출력 예측 — 포인터 기초
**문제:** 아래 코드의 출력 결과는?

```c
#include <stdio.h>
int main() {
    int x = 10;
    int *p = &x;
    *p = 20;
    printf("%d\n", x);
    return 0;
}
```

**정답:** `20`  
**해설:** `p`는 `x`의 주소를 담고 있습니다. `*p = 20`은 "p가 가리키는 곳에 20을 저장하라"는 뜻으로, 결국 `x`의 값이 바뀝니다.

---

#### [2-2] 코드 완성 — 배열 합계
**문제:** 배열의 모든 요소를 더하는 코드를 완성하세요.

```c
int sum = 0;
int arr[] = {1, 2, 3, 4, 5};
for (int i = 0; i < ___; i++) {
    sum += arr[___];
}
printf("%d\n", sum);
```

**빈칸 ①:** `5`  
**빈칸 ②:** `i`  
**출력:** `15`

## 3단계: C++ (Object-Oriented Thinking)

### 학습 목표
- 클래스와 객체로 현실 세계를 모델링하는 법을 익힌다
- 상속과 다형성으로 코드 재사용성을 높이는 사고를 기른다
- C와 C++의 차이를 통해 "왜 객체지향이 탄생했는가"를 이해한다

### 핵심 개념
1. **클래스와 객체** — 설계도와 실체
2. **멤버 변수와 메서드** — 상태와 행동
3. **접근 제어자** — `public`, `private`, `protected`
4. **생성자 / 소멸자** — 객체의 생애 주기
5. **상속 (`inheritance`)** — 코드 재사용의 핵심
6. **오버라이딩** — 부모 메서드 재정의
7. **다형성 (`polymorphism`)** — 같은 인터페이스, 다른 동작
8. **가상 함수 (`virtual`)** — 동적 바인딩
9. **참조자 (`&`)** — C++ 고유의 참조 방식
10. **STL 기초** — `vector`, `string`, `map`

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- 코드 완성 (클래스 구조 채우기)
- 출력 예측 (상속/오버라이딩 결과)
- 관계 연결 (클래스 다이어그램 매칭)

### 예시 문제

#### [3-1] 출력 예측 — 상속과 오버라이딩
**문제:** 아래 코드의 출력 결과는?

```cpp
#include <iostream>
using namespace std;

class Animal {
public:
    void speak() { cout << "..." << endl; }
};

class Dog : public Animal {
public:
    void speak() { cout << "Woof!" << endl; }
};

int main() {
    Dog d;
    d.speak();
    return 0;
}
```

**정답:** `Woof!`  
**해설:** `Dog`는 `Animal`을 상속받았지만 `speak()`를 재정의(오버라이딩)했습니다. 자식 클래스의 메서드가 우선합니다.

---

#### [3-2] 개념 퀴즈 — 클래스 vs 구조체
**문제:** C++에서 `class`와 `struct`의 기본 차이는?

> ① `struct`는 메서드를 가질 수 없다  
> ② `class`의 기본 접근 제어자는 `private`, `struct`는 `public`이다  
> ③ `class`는 상속이 불가능하다  
> ④ `struct`는 C++에서 사용할 수 없다

**정답:** ②  
**해설:** 기능적으로는 거의 동일하지만, 기본 접근 제어자가 다릅니다. 관례적으로 단순 데이터 묶음엔 `struct`, 동작 중심엔 `class`를 씁니다.

---

## 4단계: 공통 개념 (Data Structures & Algorithms)

### 학습 목표
- 언어에 무관한 핵심 자료구조를 이해하고 선택 기준을 익힌다
- Big-O 표기법으로 알고리즘 효율을 분석하는 눈을 기른다
- "어떤 도구를 언제 쓸 것인가"를 판단하는 사고력을 기른다

### 핵심 개념
1. **배열 vs 연결 리스트** — 메모리 연속성의 트레이드오프
2. **스택 (Stack)** — LIFO 구조와 활용 (함수 호출, 괄호 검사)
3. **큐 (Queue)** — FIFO 구조와 활용 (BFS, 대기열)
4. **트리 (Tree)** — 계층 구조, 이진 트리
5. **해시 테이블** — O(1) 탐색의 원리
6. **정렬 알고리즘** — 버블, 선택, 삽입, 퀵, 머지
7. **탐색 알고리즘** — 선형 탐색 vs 이진 탐색
8. **재귀 (Recursion)** — 자기 참조와 기저 사례
9. **Big-O 표기법** — O(1), O(n), O(log n), O(n²)
10. **그래프 기초** — DFS, BFS 개념

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- Big-O 분류 (드래그)
- 알고리즘 추적 (단계별 배열 상태 예측)
- 자료구조 선택 문제 ("이 상황엔 무엇을 쓸까?")

### 예시 문제

#### [4-1] 알고리즘 추적 — 버블 정렬
**문제:** 버블 정렬을 `[5, 3, 1, 4]`에 1회 패스 적용한 결과는?

**정답:** `[3, 1, 4, 5]`  
**해설:** 인접한 두 원소를 비교해 큰 것을 뒤로 보냅니다. 1회 패스 후 가장 큰 수인 5가 맨 끝으로 이동합니다.

---

#### [4-2] 개념 퀴즈 — Big-O 선택
**문제:** 정렬된 배열에서 특정 값을 찾을 때 가장 효율적인 알고리즘과 그 시간 복잡도는?

> ① 선형 탐색 — O(n)  
> ② 이진 탐색 — O(log n)  
> ③ 버블 정렬 후 탐색 — O(n²)  
> ④ 해시 탐색 — O(1)

**정답:** ②  
**해설:** 이미 정렬된 배열이라면 이진 탐색으로 절반씩 줄여가며 찾을 수 있어 O(log n)입니다. 해시는 O(1)이지만 배열이 아닌 해시 테이블이 필요합니다.

## 5단계: Python

### 학습 목표
- Python의 간결한 문법으로 "아이디어를 빠르게 코드화"하는 능력을 기른다
- C에서 배운 개념이 Python에서 어떻게 추상화되는지 비교하며 이해한다
- Pyodide를 통해 브라우저에서 직접 실행하며 즉각적 피드백을 경험한다

### 핵심 개념
1. **Python 특징** — 인터프리터, 동적 타입, 들여쓰기
2. **기본 자료형** — `int`, `float`, `str`, `bool`
3. **리스트 (List)** — 인덱싱, 슬라이싱, 메서드
4. **튜플 (Tuple)** — 불변 시퀀스
5. **딕셔너리 (Dictionary)** — 키-값 쌍
6. **집합 (Set)** — 중복 제거, 집합 연산
7. **조건문** — `if / elif / else`
8. **반복문** — `for`, `while`, `range()`
9. **함수** — `def`, 기본값 인자, 반환값
10. **리스트 컴프리헨션** — `[x for x in ...]`
11. **클래스 기초** — `__init__`, `self`
12. **모듈과 import** — 표준 라이브러리 활용

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- 코드 완성 (빈칸 채우기)
- 출력 예측 (단답형)
- 실행 확인 (**Pyodide로 직접 실행** ← 이 단계의 핵심)

### 예시 문제

#### [5-1] 출력 예측 — 리스트 슬라이싱
**문제:** 아래 코드의 출력 결과는?

```python
nums = [10, 20, 30, 40, 50]
print(nums[1:4])
```

**정답:** `[20, 30, 40]`  
**해설:** 슬라이싱 `[1:4]`는 인덱스 1부터 3까지(4 미포함)를 가져옵니다.

---

#### [5-2] 실행 확인 — 딕셔너리 활용
**문제:** 다음 코드를 완성하여 학생 점수의 평균을 출력하세요.

```python
scores = {"Alice": 90, "Bob": 75, "Carol": 85}
total = sum(___.values())
avg = total / len(___)
print(f"평균: {avg}")
```

**빈칸:** `scores`  
**출력:** `평균: 83.33333333333333`  
**실행:** Pyodide로 직접 확인 가능

## 6단계: 응용 트랙 (Applied Tracks)

> 5단계를 완료한 학습자가 목적에 따라 선택합니다.

### 트랙 A: 웹 개발
**목표:** HTML/CSS/JS로 기초 웹 페이지를 만들고 Python Flask로 서버를 구성한다  
**핵심 개념:** DOM, 이벤트, HTTP 요청, REST API 기초  
**퍼즐 예시:** HTML 구조 완성, JS 이벤트 핸들러 빈칸 채우기

### 트랙 B: 데이터 분석
**목표:** NumPy, Pandas로 데이터를 정제하고 Matplotlib으로 시각화한다  
**핵심 개념:** DataFrame, 결측치 처리, 통계 기초, 시각화  
**퍼즐 예시:** Pandas 코드 완성, 그래프 유형 선택

### 트랙 C: 시스템 프로그래밍
**목표:** C/C++ 기반 시스템 수준 프로그래밍과 OS 개념을 연결한다  
**핵심 개념:** 프로세스/스레드, 파일 시스템, 소켓 기초  
**퍼즐 예시:** 멀티스레드 경쟁 조건 예측, 시스템 콜 매칭

### 트랙 D: 게임 개발
**목표:** Pygame 또는 JS Canvas로 간단한 게임 루프를 구현한다  
**핵심 개념:** 게임 루프, 충돌 감지, 스프라이트, 상태 머신  
**퍼즐 예시:** 게임 루프 순서 배열, 충돌 조건 코드 완성

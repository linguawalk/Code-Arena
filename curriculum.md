# Code Arena — Curriculum Design

**"언어를 배우는 것이 아니라 생각하는 방법을 배운다"**
> 사이트: code-arena.org
> 최종 업데이트: 2026-07-17
> 구조: 공통 기초(순차) → 카테고리 트랙(분기 선택)

---

## 개요

Code Arena는 두 단계로 구성됩니다.

1. **공통 기초 (Foundation)** — 모든 학습자가 순서대로 거치는 구간. 컴퓨팅 사고와 하나의 "온보딩 언어(C)"를 통해 프로그래밍의 기본 감각을 익힙니다.
2. **카테고리 트랙 (Tracks)** — 기초를 마친 학습자가 관심사에 따라 선택하는 구간. 하나의 트랙 안에 여러 언어가 순차적으로 배치되어 있습니다.

이 구조는 하버드 CS50이 Scratch → C라는 공통 입문 과정을 모두에게 거치게 한 뒤, Python/AI/웹개발/보안 등 후속 과정으로 학습자를 분기시키는 방식과 동일한 원리입니다. 처음부터 10여 개 언어를 병렬로 늘어놓지 않고, 공통 기초에서 프로그래밍 사고력을 검증한 뒤 트랙으로 나누는 편이 완주율과 콘텐츠 설계 효율 모두에서 유리합니다.

| 구간 | 이름 | 핵심 사고 방식 | 예상 학습량 |
|---|---|---|---|
| 기초 0단계 | 컴퓨터 사고 기초 | 순차적으로 생각하기 | 퍼즐 30개 |
| 기초 1단계 | C 언어 (온보딩) | 컴퓨터의 언어로 생각하기 | 퍼즐 60개 |
| — | **[분기점] 트랙 선택** | — | — |
| 트랙 A | 시스템/저수준 | 메모리와 자원을 통제하며 생각하기 | 언어당 40~60개 |
| 트랙 B | 객체지향/엔터프라이즈 | 구조와 역할로 나누어 생각하기 | 언어당 40~60개 |
| 트랙 C | 스크립트/실무 | 간결하고 빠르게 생각하기 | 언어당 40~60개 |
| 트랙 D | 레거시/역사 | 과거의 제약 속에서 생각하기 | 언어당 25~35개 |
| 공통 선택 | 자료구조·알고리즘 | 효율적으로 분류하기 | 퍼즐 40개 |
| 응용 트랙 | 목적별 심화 | 목적에 맞게 생각하기 | 트랙별 30개+ |

## 기초 0단계: 컴퓨터 사고 기초 (Computer Thinking)

### 학습 목표
- 문제를 단계별로 분해(decomposition)하는 능력을 기른다
- 알고리즘이 "정확한 순서의 명령 목록"임을 이해한다
- 변수, 조건, 반복이라는 세 가지 핵심 개념을 언어 없이 체득한다

### 핵심 개념
1. **알고리즘** — 문제 해결 절차의 정의
2. **순서도 (Flowchart)** — 조건 분기와 흐름 시각화
3. **변수** — 이름이 붙은 기억 공간
4. **조건문** — "만약 ~라면" 분기
5. **반복문** — 패턴의 반복과 종료 조건
6. **함수 개념** — 재사용 가능한 명령 묶음
7. **입력 / 출력** — 컴퓨터와의 대화 구조
8. **디버깅 사고** — 오류를 찾아내는 논리

### 퍼즐 유형
- 개념 퀴즈 (객관식)
- 순서 배열 (드래그) ← 이 단계의 핵심
- 순서도 완성 (빈칸 채우기)

### 예시 문제

**[F0-1] 순서 배열 — 라면 끓이기 알고리즘**

문제: 다음 단계를 올바른 순서로 배열하세요.
> A. 물이 끓으면 면을 넣는다 / B. 냄비에 물을 붓는다 / C. 면이 익으면 스프를 넣는다 / D. 불을 켠다 / E. 그릇에 담는다

정답: B → D → A → C → E
해설: 컴퓨터 프로그램도 이처럼 순서가 틀리면 원하는 결과가 나오지 않습니다.

---

**[F0-2] 개념 퀴즈 — 변수의 역할**

문제: 다음 중 '변수'를 가장 잘 설명한 것은?
> ① 컴퓨터가 실행하는 명령어 ② 이름이 붙은 기억 공간으로, 값을 저장하고 바꿀 수 있다 ③ 프로그램의 시작점 ④ 오류가 발생하는 코드 줄

정답: ②
해설: 변수는 우편함과 같습니다. 이름은 그대로지만 안의 내용(값)은 바꿀 수 있습니다.

## 기초 1단계: C 언어 (온보딩 언어)

### 학습 목표
- 메모리와 CPU가 실제로 어떻게 동작하는지 코드로 이해한다
- 포인터를 통해 "주소로 생각하기"를 체득한다
- 이후 어떤 트랙을 선택하더라도 통용되는 프로그래밍의 기초 문법 감각을 갖춘다

> C를 공통 온보딩 언어로 두는 이유: 트랙 B(객체지향)와 트랙 C(스크립트)로 갈라져도 조건문·반복문·함수·자료형 같은 핵심 문법 감각은 C에서 익힌 것이 그대로 통용됩니다. 트랙 A(시스템/저수준)를 선택하지 않는 학습자에게도 "언어가 왜 이런 모습인지"에 대한 배경지식을 제공합니다.

### 핵심 개념
1. **컴파일 과정** — 소스코드 → 기계어 변환
2. **자료형** — `int`, `char`, `float`, `double`과 메모리 크기
3. **연산자** — 산술, 비교, 논리, 비트 연산
4. **조건문** — `if / else if / else`, `switch`
5. **반복문** — `for`, `while`, `do-while`
6. **배열** — 연속된 메모리 블록
7. **함수** — 선언, 정의, 호출, 반환값
8. **포인터** — 주소와 역참조 (`*`, `&`)
9. **문자열** — `char` 배열과 null terminator

> 구조체·동적 메모리·파일 I/O 등 심화 개념은 기초 1단계에서 다루지 않고, 트랙 A(시스템/저수준) 진입 후 이어서 학습합니다. 기초 구간은 "온보딩"이 목적이므로 과부하를 주지 않습니다.

### 퍼즐 유형
- 개념 퀴즈 (객관식) / 코드 완성 (빈칸 채우기) / 출력 예측 (단답형) / 순서 배열 (함수 실행 흐름)

### 예시 문제

**[F1-1] 출력 예측 — 포인터 기초**

```c
#include <stdio.h>
int main() {
    int x = 10;
    int *p = &x;
    *p = 20;
    printf("%d\n", x);
    return 0;
}
```

정답: `20`
해설: `p`는 `x`의 주소를 담고 있습니다. `*p = 20`은 "p가 가리키는 곳에 20을 저장하라"는 뜻으로, 결국 `x`의 값이 바뀝니다.

## [분기점] 트랙 선택

기초 1단계(C 언어)를 마치면 학습자는 4개 카테고리 트랙 중 하나 이상을 선택합니다. 트랙은 순서 제약 없이 자유롭게 선택·병행할 수 있습니다.

| 트랙 | 다루는 언어(순서) | 성격 |
|---|---|---|
| A. 시스템/저수준 | C(심화) → C++ → Rust | 메모리 관리, 포인터, 성능 중심 |
| B. 객체지향/엔터프라이즈 | Visual Basic → C# → Java | 클래스, 인터페이스, 기업 환경 |
| C. 스크립트/실무 | Python → JavaScript → PHP | 빠른 개발, 실용 문법 |
| D. 레거시/역사 | BASIC(초기형) → Pascal → Fortran → COBOL | 소수 방문자 대상, 역사적·교육적 가치 |

각 트랙 내 언어별 상세 커리큘럼(레벨 구성, 문제 유형, JSON 스키마)은 `schema/question_schema.json`과 `content/{category}/{language}/` 하위 파일을 따릅니다.

### 트랙 A: 시스템/저수준 (System / Low-level)

**학습 목표**: 메모리와 CPU가 실제로 어떻게 동작하는지 코드로 이해하고, 포인터·구조체·동적 메모리로 "주소로 생각하기"를 심화한다.

**추가 핵심 개념** (기초 1단계 이후 이어지는 내용): 구조체(`struct`), 동적 메모리(`malloc`, `free`), 파일 I/O, 이후 C++의 클래스/상속/다형성, Rust의 소유권 모델

### 트랙 B: 객체지향/엔터프라이즈 (OOP / Enterprise)

**학습 목표**: 클래스와 객체로 현실 세계를 모델링하는 법을 익히고, 상속·다형성으로 코드 재사용성을 높이는 사고를 기른다.

**핵심 개념**: 클래스와 객체, 멤버 변수/메서드, 접근 제어자, 생성자/소멸자, 상속, 오버라이딩, 다형성, 인터페이스

### 트랙 C: 스크립트/실무 (Scripting / Practical)

**학습 목표**: 간결한 문법으로 아이디어를 빠르게 코드화하는 능력을 기르고, C에서 배운 개념이 고수준 언어에서 어떻게 추상화되는지 비교한다.

**핵심 개념**: 동적 타입, 리스트/딕셔너리, 리스트 컴프리헨션, 클래스 기초, 모듈/패키지
**실행 환경**: Python은 Pyodide를 통해 브라우저에서 직접 실행하며 즉각적 피드백 제공 (트래픽 증가 후 도입)

### 트랙 D: 레거시/역사 (Legacy / Historical)

**학습 목표**: 오늘날의 언어가 왜 지금의 모습을 갖게 되었는지, 과거 언어들의 제약과 설계를 통해 이해한다.

**성격**: 실무 활용도보다 역사적 맥락과 개념적 재미에 초점. 레벨 3~4단계로 축소 운영.

## 공통 선택 모듈: 자료구조·알고리즘 (Data Structures & Algorithms)

특정 트랙에 종속되지 않는 언어 무관 개념이므로, 어느 트랙을 선택했든 관계없이 열람 가능한 공통 선택 모듈로 둡니다.

### 학습 목표
- 언어에 무관한 핵심 자료구조를 이해하고 선택 기준을 익힌다
- Big-O 표기법으로 알고리즘 효율을 분석하는 눈을 기른다

### 핵심 개념
1. 배열 vs 연결 리스트 2. 스택(Stack) 3. 큐(Queue) 4. 트리(Tree) 5. 해시 테이블
6. 정렬 알고리즘 (버블/선택/삽입/퀵/머지) 7. 탐색 알고리즘 8. 재귀 9. Big-O 표기법 10. 그래프 기초(DFS/BFS)

### 퍼즐 유형
개념 퀴즈, Big-O 분류(드래그), 알고리즘 추적(단계별 배열 상태 예측), 자료구조 선택 문제

### 예시 문제

**[DS-1] 알고리즘 추적 — 버블 정렬**

문제: 버블 정렬을 `[5, 3, 1, 4]`에 1회 패스 적용한 결과는?
정답: `[3, 1, 4, 5]`
해설: 인접한 두 원소를 비교해 큰 것을 뒤로 보냅니다. 1회 패스 후 가장 큰 수인 5가 맨 끝으로 이동합니다.

## 응용 트랙 (Applied Tracks)

> 카테고리 트랙(A~D) 중 하나 이상을 어느 정도 진행한 학습자가 목적에 따라 선택하는 심화 구간입니다. 사전 권장 트랙을 함께 표기합니다.

| 응용 트랙 | 목표 | 사전 권장 트랙 | 핵심 개념 |
|---|---|---|---|
| 웹 개발 | HTML/CSS/JS로 웹 페이지 제작, Python Flask로 서버 구성 | 트랙 C (스크립트) | DOM, 이벤트, HTTP 요청, REST API 기초 |
| 데이터 분석 | NumPy, Pandas로 데이터 정제, Matplotlib 시각화 | 트랙 C (스크립트) | DataFrame, 결측치 처리, 통계 기초 |
| 시스템 프로그래밍 | C/C++ 기반 시스템 수준 프로그래밍과 OS 개념 연결 | 트랙 A (시스템/저수준) | 프로세스/스레드, 파일 시스템, 소켓 기초 |
| 게임 개발 | Pygame 또는 JS Canvas로 간단한 게임 루프 구현 | 트랙 A 또는 C | 게임 루프, 충돌 감지, 스프라이트, 상태 머신 |

## 퍼즐 유형 상세 정의

| 유형 | 코드명 | 설명 | 사용 구간 |
|---|---|---|---|
| 개념 퀴즈 | `predict_output` / `quiz` | 4지선다 객관식 | 전 구간 |
| 코드 완성 | `fill_blank` | 빈칸 채우기 (인라인 입력) | 기초 1단계 ~ 모든 트랙 |
| 출력 예측 | `predict_output` | 코드 실행 결과 예측 | 기초 1단계 ~ 모든 트랙 |
| 에러 찾기 | `error_finding` | 문법/논리 오류 위치 찾기 | 기초 1단계 ~ 모든 트랙 |
| 개념 매칭 | `concept_matching` | 용어-설명 짝짓기 | 전 구간 |
| 순서 배열 | `order` | 드래그앤드롭 순서 정렬 | 기초 0~1단계 |
| 실행 확인 | `run` | 코드 수정 후 Pyodide/JS 직접 실행 | 트랙 C (Python), 응용 트랙 |

> `predict_output`, `fill_blank`, `error_finding`, `concept_matching` 4종은 `schema/question_schema.json`에 정식 정의되어 있습니다. `order`, `run` 유형은 기초 구간과 실행 환경 도입 이후를 위한 확장 유형으로, 스키마에 추가 예정입니다.

## 문제 ID 체계

기초 구간과 트랙 구간의 ID 규칙을 분리합니다.

```
[기초 구간 — 순차 진행]
foundation_{stage}_{unit}_{seq}
예시:
  foundation_0_02_003  → 기초 0단계, 2번 단원, 3번 문제
  foundation_1_05_012  → 기초 1단계(C), 5번 단원, 12번 문제

[트랙 구간 — 언어별]
{language}_l{level}_u{unit}_q{seq}
예시:
  c_l1_u01_q001         → 트랙A C, Level 1, 1번 단원, 1번 문제
  visual_basic_l1_u01_q001 → 트랙B Visual Basic, Level 1, 1번 단원, 1번 문제
```
